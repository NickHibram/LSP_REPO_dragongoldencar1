The Template Method pattern is used in the Report class through the generateReport() method, which defines a fixed workflow: loadData → formatHeader → formatBody → formatFooter. This method is final so subclasses cannot change the order of execution. StudentReport and CourseReport extend Report and implement the abstract methods to provide their own specific data and formatting. The parent class handles the overall structure (printing headers, body, and footer), while the subclasses only focus on the report-specific content. Polymorphism is shown in the Driver by using a List<Report> and calling generateReport() on each object.
